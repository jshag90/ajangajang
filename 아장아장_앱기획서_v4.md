## 2. 단계별 개발 전략

> **핵심 전략:** 체크리스트로 충성 사용자 확보 → 커뮤니티로 락인(Lock-in)
> 
> 커뮤니티는 사용자 기반이 쌓여야 의미가 있습니다. 1단계에서 체크리스트 핵심 기능을 완성도 높게 출시하고, MAU가 안정적으로 성장한 뒤 2단계에서 커뮤니티를 도입합니다.

### 🏗️ 1단계 — 체크리스트 MVP

> **개발 우선순위: 체크리스트 핵심 기능을 가장 먼저 완성한다**

- ✅ **[최우선] 12단계 발달 체크리스트 전체 구현**
- ✅ **[최우선] 4개 발달 영역 체크 화면 + 결과 화면**
- ✅ 아이 프로필 등록·관리
- ✅ 체크 결과 저장 (Room)
- ✅ PDF 저장 + 카카오톡 공유
- ✅ 성장 기록 그래프
- ✅ 검진 알림 (WorkManager)
- ✅ AdMob 광고 + 프리미엄 구독
- ✅ Firebase Analytics

**목표: MAU 5,000명 달성 후 2단계 진입**

### 🚀 2단계 — 커뮤니티 도입

- 🔜 같은 개월 수 부모 익명 게시판
- 🔜 카카오/구글 소셜 로그인
- 🔜 개월 수 태그 기반 글 필터링
- 🔜 익명 닉네임 자동 부여
- 🔜 전문가 Q&A 기능
- 🔜 커뮤니티 내 네이티브 광고
- 🔜 인기글 Push 알림
- 🔜 신고·차단·운영 시스템

**목표: DAU 30% 향상 + 커뮤니티 락인**

## 3. 1단계 — 체크리스트 앱 기능 명세

### 3.1 앱 화면 구조 (1단계)

```
📱 앱 시작 → 스플래시 → 온보딩(최초 1회) → 홈

🏠 홈 탭
  ├── 아이 프로필 카드 (이름·개월 수·사진)
  ├── 현재 연령 기준 권장 체크리스트 자동 추천 배너
  ├── 최근 체크 기록 요약 카드
  ├── 검진 D-Day 배너
  └── 하단 AdMob 배너 광고 (무료 사용자)

✅ 체크리스트 탭
  ├── 개월 수 선택 (12단계 가로 스크롤)
  ├── 4개 발달 영역 탭 + 진행률 표시
  ├── 항목별 체크박스 + 체크 완료 애니메이션
  └── 결과 화면 → 저장 / 공유 기능

📊 기록 탭
  ├── 월별 체크 기록 타임라인
  ├── 영역별 레이더 차트 + 성장 추이 그래프
  └── 이전 체크 대비 변화 비교

💡 발달 정보 탭
  ├── 연령별 양육 팁 아티클
  ├── 발달 이정표 인포그래픽
  └── FAQ (자주 묻는 질문)

⚙️ 설정 탭
  ├── 아이 프로필 관리 (등록·수정·삭제)
  ├── 알림 설정
  ├── 프리미엄 구독 관리
  └── 언어 설정 (한국어 / 영어)
```

### 3.2 저장 및 공유 기능 명세 ★

> 💾 소아과 진료 전 체크 → 의사에게 결과 공유 / 배우자나 조부모에게 카카오톡으로 전달 — 이 시나리오가 메인 사용 흐름입니다.

#### ① PDF 저장

| 항목 | 내용 |
|------|------|
| 생성 내용 | 아이 이름·나이, 체크 날짜, 4개 영역별 달성률, 미충족 항목 목록, 의사 공유 질문 목록, 발달 촉진 팁 |
| 구현 방법 | Android PdfDocument API → MediaStore에 저장 |
| 무료 사용자 | 보상형 광고 1회 시청 후 저장 가능 |
| 프리미엄 사용자 | 광고 없이 무제한 저장 |
| 파일명 형식 | `아장아장_홍길동_12개월_20250101.pdf` |

#### ② 이미지 저장 (갤러리)

| 항목 | 내용 |
|------|------|
| 생성 내용 | 체크 결과 요약 카드 이미지 1장 (영역별 달성률 + 아이 이름 + 날짜) |
| 구현 방법 | Compose Canvas 비트맵 렌더링 → MediaStore Images에 저장 |
| 무료 사용자 | 보상형 광고 1회 시청 후 저장 가능 |
| 프리미엄 사용자 | 광고 없이 무제한 저장 |
| 활용 시나리오 | 인스타·카카오스토리 공유, 육아 일기 첨부 |

#### ③ 외부 공유 (Share)

| 공유 방식 | 내용 | 무료/프리미엄 |
|-----------|------|--------------|
| 카카오톡 공유 | 결과 이미지 + 앱 다운로드 링크 자동 첨부 (바이럴 효과) | 무료 제공 |
| 이메일 공유 | PDF 첨부 + 의사 공유용 텍스트 자동 생성 | 무료 제공 |
| 기타 앱 공유 | AndroidX ShareCompat — 모든 앱으로 공유 | 무료 제공 |
| 링크 공유 | 체크 결과 웹 뷰 링크 (Firebase Dynamic Links) | 프리미엄 전용 |

> **📌 공유 UX 핵심 원칙**
> - 공유 기능은 무료로 열어두기 — 공유 시 앱 링크 자동 삽입으로 바이럴 유도
> - PDF/이미지 저장은 보상형 광고와 연결 — 사용자 능동 선택, 강제 X
> - 카카오톡 공유 메시지 하단에 항상 '아장아장 앱으로 확인하기' 링크 삽입
> - 저장 완료 시 토스트 + 갤러리 열기 버튼 제공

### 3.3 체크리스트 화면 컴포넌트 명세

| 컴포넌트 | 기능 설명 | 구현 기술 |
|---------|-----------|----------|
| 개월 수 선택 | LazyRow 가로 스크롤, 선택 단계 강조 + 아이 현재 나이 자동 선택 | LazyRow |
| 영역별 탭 | 4개 발달 영역 TabRow, 각 탭에 달성률 배지 표시 | TabRow + Badge |
| 체크 항목 카드 | 항목별 체크박스 + 설명 텍스트, 체크 시 애니메이션 | Lottie |
| 진행률 바 | 현재 영역 + 전체 진행률 LinearProgressIndicator | LinearProgress |
| 결과 보기 버튼 | 전체 체크 완료 후 활성화, 미완료 시 경고 다이얼로그 | Button |
| 전면 광고 | 결과 확인 진입 시 1회 노출 (프리미엄 제외) | AdMob Interstitial |

### 3.4 결과 화면 컴포넌트 명세

| 컴포넌트 | 기능 설명 | 무료/프리미엄 |
|---------|-----------|--------------|
| 영역별 달성률 차트 | 4개 영역 도넛/레이더 차트 시각화 | 무료 |
| 미충족 항목 목록 | 빨간색 강조 + '의사와 상담하세요' 경고 | 무료 |
| 의사 공유 질문 목록 | 진료 시 확인할 질문 체크리스트 | 무료 |
| 발달 촉진 팁 카드 | 연령별 맞춤 양육 팁 5~10개 | 무료 |
| PDF 저장 버튼 | 보상형 광고 시청 후 저장 / 프리미엄 무제한 | 광고 or 프리미엄 |
| 이미지 저장 버튼 | 결과 카드 이미지 갤러리 저장 | 광고 or 프리미엄 |

## 4. 2단계 — 커뮤니티 기획

### 4.1 커뮤니티 도입 조건

2단계 진입 기준 (모두 충족 시 개발 시작)

- MAU 5,000명 이상 안정적 유지
- 앱 평점 4.0점 이상 (Play Store)
- 체크리스트 주요 버그 0건 상태
- 커뮤니티 운영 정책(신고·제재 기준) 사전 수립 완료

### 4.2 커뮤니티 핵심 컨셉

> **"지금 딱 우리 아이랑 같은 시기 부모들만 모이는 공간"**
>
> 기존 맘카페는 모든 연령이 섞여 있어 원하는 정보를 찾기 어렵습니다. 아장아장 커뮤니티는 **아이 개월 수 기반으로 자동 필터링되는 폐쇄형 구조**가 핵심 차별점입니다.

- 개월 수 태그 기반 자동 분류 — 내 아이와 같은 개월 수 글만 우선 노출
- 익명 닉네임 자동 부여 — `12개월맘_닭강정` 형태
- 소셜 로그인 필수 — 카카오 또는 구글 로그인 (어뷰징 방지)
- 발달 고민 중심 — 체크리스트 결과 공유, 발달 고민 Q&A 위주

### 4.3 사용자 인증 방식

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **소셜 로그인 (채택 ✅)** | 카카오·구글 로그인 → 내부 계정 → 게시판은 익명 | 구현 빠름, 어뷰징 차단 용이, Firebase Auth 활용 | 가입 허들 존재 |
| 기기 ID 기반 | 설치 시 자동 UUID 생성, 로그인 없이 사용 | 가입 마찰 없음 | 재설치 시 초기화, 어뷰징 취약 |
| 전화번호 인증 | 게시판 최초 이용 시 1회 본인인증 | 어뷰징 최강 차단 | 가입 허들 매우 높음 |

> **채택 방식: 소셜 로그인 + 익명 닉네임**
> - Firebase Authentication — 카카오 OAuth2 또는 Google Sign-In
> - 로그인 후 DB에 익명 닉네임 자동 부여 (개월 수 + 랜덤 단어)
> - 닉네임은 사용자가 1회 변경 가능
> - 게시판에는 닉네임만 노출, 실제 계정 정보 절대 표시 안 함
> - 백엔드: Firebase Firestore (서버 없이 구현 가능)

### 4.4 커뮤니티 화면 구성

```
💬 커뮤니티 탭 (기존 5탭 → 6탭)

📋 게시판 홈
  ├── 내 아이 개월 수 기준 인기글 상단 노출
  ├── 전체 / 개월 수별 탭 필터
  └── 글 작성 FAB 버튼

✏️ 글 작성
  ├── 개월 수 태그 자동 첨부 (내 아이 기준)
  ├── 카테고리 선택: 발달 고민 / 결과 공유 / 일상 / 질문
  └── 사진 첨부 (선택)

🔔 운영 시스템
  ├── 신고 기능 — 3회 이상 신고 시 자동 블라인드
  ├── 차단 기능 — 특정 닉네임 차단 가능
  └── 금지어 필터 — 비속어·의학 오정보 자동 감지
```

### 4.5 커뮤니티 수익화 연계

| 수익 방식 | 설명 | 예상 효과 |
|-----------|------|-----------|
| 게시판 네이티브 광고 | 피드 내 육아 관련 네이티브 광고 노출 | CTR 높음 — 타겟 정확 |
| 전문가 유료 Q&A | 인증 소아과 의사·발달치료사 답변 (건당 ₩2,000) | B2C 수익 + 신뢰도 향상 |
| 프리미엄 커뮤니티 | 글 무제한 작성, 사진 첨부 무제한 (프리미엄 전용) | 구독 전환율 향상 |
| 브랜드 스폰서 채널 | 육아 브랜드 공식 채널 운영 (월 ₩50,000~) | B2B 제휴 수익 |

## 5. 기술 스택 및 아키텍처

### 5.1 전체 기술 스택

| 계층 | 라이브러리/기술 | 1단계 | 2단계 |
|------|----------------|-------|-------|

```kotlin
// 아이 프로필
@Entity(tableName = "child_profile")
data class ChildProfile(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val name: String,
    val birthDate: LocalDate,
    val gender: String,           // "MALE" | "FEMALE"
    val photoUri: String? = null,
    val isPrimary: Boolean = false
)

// 체크 기록
@Entity(tableName = "check_record")
data class CheckRecord(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val childId: Int,             // FK → ChildProfile
    val ageMonths: Int,
    val checkDate: LocalDate,
    val socialScore: Float,       // 사회적/감정적 달성률 (0.0 ~ 1.0)
    val languageScore: Float,     // 언어/의사소통 달성률
    val cognitiveScore: Float,    // 인지력 달성률
    val motorScore: Float,        // 움직임/신체발달 달성률
    val checkedItems: String,     // JSON — 체크된 항목 ID 목록
    val notes: String? = null,
    val pdfSavedAt: LocalDateTime? = null
)
```
